Part of #563. Add a JSON output mode for the doc generator so that downstream tooling can consume documentation as data.

## Scope

- `axiomc doc --json` emits `{ functions: [...], types: [...] }` per package.
- Output is generated by the AxiOM doc generator from K.1, not by Rust.
- Schema added to `stage1/schemas/` and validated in tests like the existing v1 envelope.

## Acceptance

- `axiomc doc --json` output validates against the new schema.
- Schema validation test runs in CI under the same harness as `cli_json_outputs_validate_against_public_v1_schema`.

## Depends on

- Phase-K.1 (`std/doc.ax` exists).

## Out of scope

- Markdown output flag — K.3.
