Part of #563. Add a Markdown output flag and validate that the rendered output is byte-correct against golden fixtures.

## Scope

- `axiomc doc --md <path>` writes Markdown files into the project's `dist/docs/` directory.
- Markdown output includes function signatures, doc comments, and module summaries (one section per `.ax` file).
- Output is generated by the AxiOM doc generator from K.1.

## Acceptance

- `axiomc doc --md stage1/examples/hello` produces output bytes-equal to a checked-in golden Markdown file.
- A property test in `std/doc.ax`'s test suite asserts that round-tripping a `///` comment through `extract → render` preserves its text.

## Depends on

- Phase-K.1 (`std/doc.ax`).
- Phase-I.3 (property runner) for the round-trip property test.

## Out of scope

- Theming / templating — independent follow-up.
