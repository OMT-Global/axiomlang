# Stage1 LSP analyzer

`axiomc lsp` starts the bounded `axiom-analyzer` Language Server Protocol
endpoint over stdio.

This is the first accepted #1461 editor slice. It retains bounded, versioned
workspace state and exposes only results derived from AxiOM package and
semantic data:

- JSON-RPC 2.0 messages framed with `Content-Length`.
- `initialize`, `initialized`, `shutdown`, and `exit`.
- `textDocument/didOpen` and version-checked incremental
  `textDocument/didChange`; stale versions and invalid ranges are rejected
  without replacing the last valid in-memory text.
- `textDocument/publishDiagnostics` for each indexed workspace document,
  generated by the stage1 parser, HIR type checker, capability checks, and MIR
  lowering path using the containing package's manifest capabilities.
- `textDocument/hover`, `textDocument/definition`,
  `textDocument/references`, document/workspace symbols, completion, and
  signature help over the persistent package-aware semantic index.
- `$/cancelRequest` and `axiom/serverStatus`, which report cancellation,
  workspace generation, semantic nodes, package metadata, capability names,
  bounded memory, latest indexing latency, and a bounded-window p95 indexing
  latency measurement.
- Diagnostic ranges use additive `end_line` / `end_column` fields when the
  compiler can identify a full source range; point-only diagnostics still fall
  back to a one-character LSP range.

Unsupported request methods receive JSON-RPC `-32601`.

## Running

```bash
cargo run --manifest-path stage1/Cargo.toml -p axiomc -- lsp
```

Editor clients should configure `axiomc lsp` as a stdio language server for
`.ax` files.

## Bounds and follow-up scope

The index has a 1 MiB per-document limit, a 16 MiB total source limit, and a
2,048-document cap. On disk it indexes only `.ax` files below configured
workspace roots and skips generated, VCS, and symlinked paths. Open editor
documents always override their on-disk equivalents, including while a file is
temporarily incomplete; a declaration-header fallback preserves navigation in
that state.

Rename, code actions, formatting, semantic tokens, and long-running request
interruption remain separate accepted slices. The server does not make edits,
open listeners, or grant capabilities; it is a bounded stdio analysis service.
